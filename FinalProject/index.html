Introduction

For this final assignment we will be taking a look at what a function is, how it is structured and build a working function for a private contractor that sells mechandise online. For each product that is sold the contractor pays half of what is made on each item, then with the other half, thirty percent must be withheld for taxes, ten percent for savings, and finaly ten percent for charity.

The function must be able to take in the input values from a form and calculate how the money is to be devided and how much is left over for profit.

What is a function

A function is a block of code that performs an action or returns a value. Functions make reusing code easier so that you dont have to keep re-writing out the same code when it's needed, which makes your program more modular and efficient.

Defining a Function

There are two different ways to define a function in JavaScript:

A Function Declaration defines a named function.

A named function is a set of related statements that is assigned a name. You can use this name to reference, or call, this set of statements in other parts of your code.

—JavaScript The Web Warrior Series pg.74

The syntax for defining a named function is:

      
        function name (parameters) {
          statements;
        }

Named functions are generally used when you want to reuse the function statements within your code.

A Function Expression defines a named or anonymous function.

An anonymous function, . . . is a set of related statements with no name assigned to it. The statements in an anonymous function work only in a single context—the place in the code where they are located. You cannot reference an anonymous function anywhere else in your code.

—JavaScript The Web Warrior Series pg.74

The syntax for defining an anonymous function is the same as a named function exept that it does not specify a name:

      
        function (parameters) {
          statements;
        }

Anonymous function are used for statements that you need to run only once.

For what we are going to do, we will use a named function. Let's define and name our function.

      
        divideSaleTotals (parameters) {
          statements;
        }

Our named function now has the name divideSaleTotals. We are still missing the parameters and statements that will make our function, function.

Our function actually looks like this:

      
        divideSaleTotals () {

        }

Parameters vs. Arguments

Parameters

Parameters are placed within the parentheses that follow a function name. A parameter is a variable that is used within a function. Placing a parameter name within the parentheses of a function definition is the equivalent of declaring a new variable.

—JavaScript The Web Warrior Series pg.75

Here's an example with two parameters - parameter1 & parameter2:

      
        const parameter1 = true;
        const parameter2 = false;

        funtion twoParameters (parameter1, parameter2) {
          console.log(parameter1, parameter2);
        }

Arguments

Arguments are the values the function receives from each parameter when the function is executed (invoked). In the above example, our two arguments are true & false

For our function we only have one perameter that takes in one argument. The sales - item total.

      
        divideSaleTotals (itemTotal) {
          statements;
        }

Function statements

Following the parentheses that contain the function parameters is a set of braces (called function braces) that contain the function statements. Function statements are the statements that do the actual work of the function,

—JavaScript The Web Warrior Series pg.75

Our particular function needs statements to figure out the items cost (which is half of the sales price), then from the contractors half; thirty percent needs to be calulated for tax withholdings, ten percent for savings after taxes have been taken out, and ten percent for charity leaving us with whats left over as profit.

Our statements might look something like this:

      
        divideSaleTotals (itemTotal) {
          var itemCost = itemTotal / 2;
          var taxes = itemCost * '.3';
          var tenPercent = (itemCost - taxes) * '.1';
          var profit = itemCost - taxes - (tenPercent * 2);
        }

The statements presented above calculate the values that we need to output to the user. In order to do this we need to return the values to the html document with a couple more statements.

      
        function divideSaleTotals(itemTotal) {
          ...
          var itemCostOutput = document.getElementById('itemCost');
          itemCostOutput.innerHTML = '$' + itemCost.toFixed(2);

          var taxesOutput = document.getElementById('taxes');
          taxesOutput.innerHTML = '$' + taxes.toFixed(2);

          var savingsOutput = document.getElementById('savings');
          savingsOutput.innerHTML = '$' + tenPercent.toFixed(2);

          var charityOutput = document.getElementById('charity');
          charityOutput.innerHTML = '$' + tenPercent.toFixed(2);

          var profitsOutput = document.getElementById('profits');
          profitsOutput.innerHTML = '$' + profit.toFixed(2);
          
        }

The added statements call on an html div element with a specific id to display the values the function has calulated.

Invoking a Function.

Functions execute when the function is called. This process is known as invocation. You can invoke a function by referencing the function name, followed by an open and closed parenthesis: ().

To call our function we will add onclick="function_name()" to the forms submit button that invokes our function.

      
        <form>
          ...
          <button onclick="divideSaleTotals()">Calculate</button>
        </form>

In order for this to work correctly remember that we are passing in a parameter that takes an argument from the forms input. To do this we call it between the opening and closing braces in the forms button onclick.

      
        <form>
          ...
          <button onclick="divideSaleTotals(document.getElementById('input').value)">Calculate</button>
        </form>

With all this in place our function should work correctly.
